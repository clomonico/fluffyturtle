## Purpose

Requestors should never have to guess what the process needs from them. This document provides step-by-step submission guides, complete vs. incomplete examples, and answers to the questions Joanna's team (and teams like hers) actually ask. The goal is to eliminate the "shouting into a void" experience by making requirements clear before the request is submitted.

> "I feel like I'm shouting into a void every time I submit a request." — Joanna, Department Manager

## Per-Request-Type Submission Guides

### 1. Software License — New Purchase

**What you need before you start:**
- Vendor name and product
- Annual cost (check the vendor's website or request a quote)
- Budget code (see FAQ below)
- Business justification: one sentence on why this tool is needed and what it replaces (if anything)

**Steps:**
1. Open the submission form and select "Software License — New"
2. Enter vendor name, product name, and annual cost
3. Enter your department budget code
4. Write a one-sentence business justification
5. Attach the vendor quote or pricing page screenshot
6. Submit

**What happens next:** Your request routes to your budget holder. If the cost is above $5K, Finance reviews after budget holder approval. You will receive a status notification at each step.

---

### 2. Software License — Renewal

**What you need before you start:**
- Previous approval reference number (search "My Past Requests" in the portal)
- Confirmation that vendor, terms, and cost are unchanged (or note what changed)
- Budget code

**Steps:**
1. Open the submission form and select "Software License — Renewal"
2. Enter the previous approval reference number
3. Confirm "No changes" or describe what changed
4. Submit

**Expected turnaround:** 2 business days for unchanged renewals under $5K. The system will pre-populate vendor and cost from the prior approval.

> **Why this matters:** Joanna's team renewed the same $1,200 license four years running. Each time, it went through a full approval cycle as if it were new. The renewal path eliminates that re-justification for unchanged, low-dollar renewals.

---

### 3. Vendor Contract — New

**What you need before you start:**
- Vendor name and contact
- Statement of Work or scope description
- Total contract value
- Budget code
- Vendor quote (see FAQ: "What counts as a vendor quote?")
- If over $25K: three competitive quotes (see FAQ: "Do I need three quotes?")

**Steps:**
1. Open the submission form and select "Vendor Contract — New"
2. Enter vendor details and contract value
3. Upload Statement of Work or scope description
4. Upload vendor quote(s)
5. Enter budget code and department
6. Submit

**What happens next:** Budget holder reviews first. Finance reviews for contracts over $5K. Compliance reviews for contracts over $25K or involving external data access. You will see each approver's status in your request dashboard.

---

### 4. Vendor Contract — Renewal

**What you need before you start:**
- Previous contract approval reference number
- Confirmation that scope and terms are unchanged (or a summary of changes)
- Updated vendor quote if pricing changed
- Budget code

**Steps:**
1. Open the submission form and select "Vendor Contract — Renewal"
2. Enter the previous approval reference number
3. Confirm "No changes" or describe what changed
4. If pricing changed: upload new vendor quote
5. Submit

---

### 5. Budget Reallocation

**What you need before you start:**
- Source budget code (where funds are coming from)
- Destination budget code (where funds are going)
- Amount to reallocate
- Business justification

**Steps:**
1. Open the submission form and select "Budget Reallocation"
2. Enter source and destination budget codes
3. Enter the amount
4. Write a brief justification (why the reallocation is needed)
5. Submit

**Note:** Budget reallocations always require department head and Finance approval regardless of amount.

---

### 6. Contractor / External Access

**What you need before you start:**
- Contractor name and company
- Type of access needed (system access, physical building access, shared drive)
- Duration of access
- Sponsoring manager name
- Budget code (if billable)

**Steps:**
1. Open the submission form and select "Contractor / External Access"
2. Enter contractor details and access type
3. Specify start and end dates for access
4. Confirm sponsoring manager
5. Submit

> **From Joanna's experience:** "I submitted a request for contractor access to a shared drive. Three business days go by, I hear nothing. My contractor sat idle for a week." The new form pre-routes external access requests to Security in parallel with budget holder review, cutting the sequential wait.

---

### 7. Cross-Departmental Resource Request

**What you need before you start:**
- Name of the resource or team member being requested
- Their home department and manager
- Duration and expected time commitment (e.g., 50% for 6 weeks)
- Business justification
- Budget code for any cost allocation

**Steps:**
1. Open the submission form and select "Cross-Departmental Resource"
2. Enter resource details, home department, and requesting department
3. Specify duration and time commitment
4. Write business justification
5. Submit — the system will automatically notify both department heads

> **Why the formal path matters:** Joanna's team stopped submitting these entirely because it took 23 days. The informal "handshake deals" that replaced the process leave no paper trail, which creates untracked resource allocation and financial exposure. The redesigned path targets 7 business days with parallel routing to both department heads.

## Complete vs. Incomplete Submissions — Side by Side

### Example: New Software License

| Field | Complete Submission | Incomplete Submission |
|---|---|---|
| Vendor Name | "Figma, Inc." | "Design tool" |
| Product | "Figma Enterprise — 10 seats" | "Figma" |
| Annual Cost | "$4,500/year" | "About $5K?" |
| Budget Code | "MKT-2026-OPS-4410" | *(left blank)* |
| Justification | "Replacing Sketch for UI design. Current Sketch license expires April 30. Migration planned for Q2." | "Need it for the team" |
| Vendor Quote | PDF attached from Figma sales rep | *(nothing attached)* |

**What happens with the incomplete version:** The request gets returned to you for clarification. That round-trip adds 2-3 business days. The approver can't verify cost without a quote, can't allocate budget without a code, and can't assess urgency without a justification.

### Example: Vendor Contract Renewal

| Field | Complete Submission | Incomplete Submission |
|---|---|---|
| Previous Approval Ref | "APP-2025-0342" | *(left blank — "I don't remember")* |
| Changes from Prior Year | "No changes — same scope, vendor, and pricing" | *(left blank)* |
| Updated Quote | "N/A — renewal at same rate per attached email confirmation" | *(nothing attached)* |
| Budget Code | "ENG-2026-SVC-3200" | "Same as last time" |

**What happens with the incomplete version:** Finance can't confirm pricing hasn't changed. The request routes to full new-contract review instead of the faster renewal path. "Same as last time" isn't a budget code the system can validate.

## FAQ: The Questions Requestors Actually Ask

### "Where do I find my budget code?"

Your budget code is a department-year-category-line format (e.g., `MKT-2026-OPS-4410`). Three ways to find it:

1. **Budget Portal:** Log into the Finance self-service portal → "My Department Budget" → your active budget lines are listed with codes
2. **Ask your department admin:** Every department has a budget coordinator who can look it up. If you don't know who yours is, check the Finance intranet page under "Department Contacts"
3. **Check a past approval:** If you've submitted a similar request before, open "My Past Requests" in the approval portal — the budget code from your prior submission is displayed in the request details

> **Tip:** Bookmark your most-used budget codes. Most teams use 2-3 codes for 90% of their requests.

### "What counts as a vendor quote?"

A vendor quote is a written price confirmation from the vendor that includes:
- Vendor name and contact
- Product or service description
- Price (unit cost, total cost, or both)
- Validity period (how long the quote is good for)

**Acceptable formats:** PDF quote from vendor, email from vendor sales rep with pricing, screenshot of vendor pricing page with date visible.

**Not acceptable:** Your own estimate of the cost, a verbal price mentioned in a call (unless followed up in writing), or a price from a third-party review site.

### "Do I need three quotes?"

- **Under $25K:** No. One vendor quote is sufficient.
- **$25K and above:** Yes. Three competitive quotes are required per procurement policy. This applies to new contracts only — renewals at the same or lower price do not require re-quoting.
- **Sole-source exception:** If only one vendor provides this product or service, submit a sole-source justification (one paragraph explaining why no alternatives exist). Finance will review and may waive the three-quote requirement.

### "My request is urgent. Can it be expedited?"

Yes. When creating your request, check the "Expedited Review" flag and add a brief explanation of the business impact of delay (e.g., "Contractor starts Monday and needs system access to begin billable work"). Expedited requests route in parallel to all required approvers rather than sequentially. This typically cuts turnaround by 40-60%.

**Note:** Overuse of the expedited flag will reduce its effectiveness. If more than 30% of a department's requests are flagged as expedited, the system will flag the pattern and your department head will be notified.

### "I submitted my request but I don't know what's happening. How do I check status?"

Open the approval portal → "My Requests" → click any request to see:
- Current step and approver name
- Estimated completion date
- History of all prior steps with timestamps
- Any comments or questions from approvers

You will also receive email or Slack notifications when your request moves to a new step or when an approver has a question.

### "My request was returned for more information. What do I do?"

Open the returned request in "My Requests." The approver's specific question or missing-item note will be displayed at the top. Add the requested information and click "Resubmit." Your request re-enters the queue at the same step — it does not restart from the beginning.

### "What if I don't know which request type to choose?"

The submission form includes a "Help me choose" wizard. Answer three questions (What are you requesting? Is it new or a renewal? What's the approximate cost?) and the system recommends the correct request type. If you're still unsure, select "General Request" and a routing analyst will classify it for you within one business day.

## Common Mistakes and How to Avoid Them

| Mistake | Why It Happens | How to Avoid It |
|---|---|---|
| Leaving budget code blank | "I'll add it later" or "Finance knows our code" | Look it up before starting. The form won't submit without it. |
| Submitting a renewal as a new request | Request types look similar in the dropdown | Renewals have a "Previous Approval Ref" field — if you can fill that in, you want the renewal path. |
| Entering estimated cost instead of quoted cost | Vendor quote isn't handy at the moment of submission | Get the quote first, then start the form. Estimates trigger a verification loop that adds days. |
| Skipping the justification field | "It's obvious why we need this" | It isn't obvious to the approver who reviews 30 requests a week. One sentence is enough. |
| Not attaching the vendor quote | "I'll send it separately" | Attachments that arrive separately from the request get lost. Always attach to the form. |
| Selecting "Expedited" for routine requests | "It can't hurt to check the box" | It can. Expedited flags are tracked; overuse triggers a department-level review of your request patterns. |
| Using a department credit card to avoid the process | "It's faster and under $2,500" | Untracked purchases create compliance gaps and financial exposure. The redesigned process targets same-day turnaround for sub-$2,500 low-risk requests. |

## How Guidance Integrates with the Submission Form

The guidance above is not just a document people read once and forget. It is woven into the submission experience at three levels:

### Tooltips (Field-Level Help)
Every required field in the submission form includes a tooltip icon (ℹ️). Hovering or clicking shows:
- What the field means
- Where to find the information
- An example of a good entry

Example tooltip for "Budget Code":
> *Your department budget code in DEPT-YEAR-CAT-LINE format (e.g., MKT-2026-OPS-4410). Find it in the Finance portal under "My Department Budget" or check a past approval in "My Requests."*

### Help Links (Section-Level Guidance)
Each section of the form (Vendor Details, Cost Information, Justification) includes a "Need help?" link that opens the relevant section of this guidance document in a side panel. The requestor can read the FAQ or examples without leaving the form.

### Inline Validation Messages
The form validates entries in real time and provides actionable error messages:

| Validation | Message Shown |
|---|---|
| Budget code field is blank | "Budget code is required. Find yours in the Finance portal → My Department Budget." |
| Cost entered without quote attached | "Please attach a vendor quote for the cost you entered. Acceptable formats: PDF, email screenshot, or vendor pricing page." |
| Renewal selected but no prior ref entered | "Enter your previous approval reference number. Find it in My Requests → search by vendor name." |
| Estimated cost entered (non-round number check disabled) | "Is this a quoted price or an estimate? Estimates may trigger a verification step. Attach a vendor quote to skip verification." |
| Expedited flag checked without impact statement | "Briefly describe the business impact of delay to support expedited routing." |

## Feedback Mechanism: Reporting Confusing Requirements

### During Submission
Every page of the submission form includes a "This is confusing" button in the footer. Clicking it opens a short feedback form:
- Which field or instruction was unclear? *(dropdown of fields on current page)*
- What were you trying to do? *(free text, 280 characters)*
- Submit

Feedback is reviewed weekly by the process improvement team. Patterns (e.g., 15 people report the same field is unclear in one month) trigger a guidance update within the following sprint.

### After Submission
When a request is completed (approved or returned), the requestor receives a one-question survey:
> "Was the submission process clear? Yes / Mostly / No"

Selecting "Mostly" or "No" opens a follow-up: "What was unclear?" Responses feed the same weekly review.

### Quarterly Review
Joanna's team and two other high-volume requesting departments participate in a quarterly 30-minute review of the top 5 reported friction points. This keeps the guidance alive and evolving instead of becoming stale documentation.

## Joanna's Perspective: What Requestors Actually Struggle With

Everything in this document is grounded in what Joanna and her team told us during interviews. Here are the specific pain points that shaped the guidance:

**"I don't know where my request is."**
The redesigned process includes status tracking at every step, email/Slack notifications on state changes, and a dashboard showing current approver and estimated completion. This guidance document tells requestors exactly where to find that information.

**"Why do I need to re-justify the same purchase every year?"**
The renewal path eliminates re-justification. If vendor, scope, and price haven't changed, the requestor confirms "no changes" and the request routes through a lighter review. This is reflected in the per-type guides above.

**"Nobody told me contractor access needs Finance sign-off."**
The request-type table at the top of this document and the quick-reference cards show exactly which approvers are involved for each type. No surprises mid-process.

**"We started using credit cards because the process is too slow."**
The redesigned process targets same-day turnaround for sub-$2,500 low-risk requests. The "Common Mistakes" section addresses why credit card bypass creates risk, and the expedited flag provides a legitimate fast path when urgency is real.

**"Cross-departmental requests take 23 days so we stopped submitting them."**
The cross-departmental path now routes to both department heads in parallel (not sequentially) and targets 7 business days. The submission guide walks requestors through exactly what to prepare so the request doesn't bounce back for missing information.

**"I can't plan around an unpredictable process."**
Every request type in this document includes an expected turnaround. The submission form displays an estimated completion date at the time of submission. Consistency over raw speed is the design principle.
